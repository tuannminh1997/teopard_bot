🐆 Teopard Bot - Hướng dẫn sử dụng

1. Kích hoạt tài khoản
/start - Bắt đầu bot
/whoami - Xem Telegram User ID của bạn

Nếu tài khoản chưa được kích hoạt, hãy gửi User ID cho admin để được thêm vào whitelist.

2. Phân tích coin
Gõ mã coin, ví dụ:
BTC
ETH
SOL

Sau đó chọn:
- Scalp: phân tích ngắn hạn 15M / 1H / 4H
- Swing: phân tích dài hạn 4H / 1D / 1W

3. Lệnh user
/listsymbols - Xem danh sách coin được phép phân tích
/stats - Xem thống kê win/loss của chính bạn
/stats BTC - Xem thống kê BTCUSDT của chính bạn
/history - Xem 10 dự đoán gần nhất của chính bạn
/history BTC - Xem lịch sử BTCUSDT của chính bạn
/dashboard - Xem dashboard nhanh của chính bạn
/help - Xem hướng dẫn này

Lưu ý: User thường chỉ xem được lịch sử/dashboard của chính mình, không xem được prediction của user khác.

Bot tính sẵn dữ liệu kỹ thuật trước khi gửi AI: EMA, RSI, MACD, khối lượng, ATR, cấu trúc giá, Fibonacci, vùng quét Long/Short và rủi ro tối thiểu. Claude sẽ tự phân tích và tự đặt Entry/SL/TP; Python kiểm tra lại logic trước khi lưu prediction.

Bot không còn dùng Python validator để ẩn phản hồi của Claude. Claude trả thế nào thì user thấy thế đó. Python chỉ parse tối thiểu Entry/SL/TP để lưu auto-check nếu đủ số. Nếu không parse đủ số, bot vẫn hiển thị phản hồi cho user nhưng chỉ lưu hidden record để learning/debug và không auto-check.

4. Auto-check kết quả
Scalp:
- Chờ Entry tối đa 12h
- Nếu khớp Entry, theo dõi tối đa 72h
- Check mỗi 1h
- Chấm TP/SL bằng nến 15M

Swing:
- Chờ Entry tối đa 24h
- Nếu khớp Entry, theo dõi tối đa 7 ngày
- Check mỗi 12h
- Chấm TP/SL bằng nến 1H

Bot tự động check và cập nhật DB, nhưng không gửi thông báo kết quả tự động cho user/admin nữa. User muốn xem kết quả thì chủ động dùng /history, /stats hoặc /dashboard.

5. Lệnh admin
Admin dùng lệnh user như người dùng bình thường:
/stats - Xem thống kê của chính admin
/history - Xem lịch sử của chính admin
/dashboard - Xem dashboard của chính admin

Admin muốn xem toàn hệ thống thì dùng lệnh riêng:
/statsall - Xem thống kê toàn bộ user
/statsall BTC - Xem thống kê BTCUSDT toàn bộ user
/historyall - Xem lịch sử toàn bộ user
/historyall BTC - Xem lịch sử BTCUSDT toàn bộ user
/dashboardall - Xem dashboard toàn bộ hệ thống

Lệnh quản trị:
/adduser <user_id> - Thêm user vào whitelist
/removeuser <user_id> - Xóa user khỏi whitelist
/listusers - Xem danh sách whitelist
/setlimit <user_id> <số lượt> - Set giới hạn lượt/ngày
/resetusage <user_id> - Reset lượt dùng hôm nay
/addsymbol BTC - Thêm symbol
/removesymbol BTC - Xóa symbol
/checknow - Kiểm tra prediction đến hạn ngay lập tức; chỉ cập nhật DB và báo số lượng, không gửi kết quả cho user/admin
/clearhistory CONFIRM - Xóa toàn bộ lịch sử prediction, giữ whitelist và symbol

Telegram menu không hiện lệnh có tham số, nên các lệnh như /stats BTC, /history BTC, /adduser 123456789 phải gõ tay.

Ghi chú admin: Admin dùng /historyall hoặc /historyall BTC sẽ thấy thêm User ID và Chat ID của từng prediction để biết lệnh đó thuộc user nào. /history chỉ xem lịch sử của chính admin nên không hiện User ID.

Ghi chú học lại: Khi bạn phân tích coin, Claude chỉ dùng lịch sử phân tích của chính tài khoản bạn cho cùng coin/mode. Lịch sử của user khác không được đưa vào prompt học lại. Admin vẫn xem được thống kê/lịch sử toàn hệ thống khi dùng /statsall, /historyall, /dashboardall.

Ghi chú học lại nâng cao: Lịch sử đưa cho Claude học gồm 5 phân tích gần nhất của chính bạn cho cùng coin/mode. Mỗi lịch sử có Entry/SL/TP, kết quả WIN/LOSS, lý do kết quả, reasoning cũ, market snapshot và feature snapshot gồm cấu trúc, Fibonacci, vùng quét, ATR/risk, EMA/RSI/MACD/volume.


Cập nhật format phản hồi:
- Bắt buộc có đủ các mục Thanh khoản, Quyết định, Entry/SL/TP, Kịch bản chính và Rủi ro.
- Bot không dùng cụm “swing gần/swing lớn” trong output cho user; thay bằng “đỉnh/đáy gần/biên lớn”.
- Nếu Claude trả thiếu format, Python không tự sửa nội dung; bot ẩn phản hồi đó, trả thông báo “chưa đủ dữ liệu hợp lệ để tạo phân tích” cho user và lưu hidden REJECTED_PLAN để học/debug.

V4.8 compact output guard update:
- Output cho user dùng format rút gọn: không hiện riêng “Bối cảnh” và “Cấu trúc”.
- Bot vẫn gửi dữ liệu EMA/RSI/MACD/ATR/Fibonacci/cấu trúc/vùng quét cho Claude để phân tích nội bộ.
- Prompt vẫn yêu cầu Claude có đủ Thanh khoản, Quyết định, Entry/SL/TP, Kịch bản chính và Rủi ro, nhưng Python không ẩn phản hồi nếu Claude thiếu format.


Bản cập nhật thông báo lỗi:
- Claude trả LONG/SHORT thì bot hiển thị trực tiếp. Nếu parse đủ Entry/SL/TP thì bot lưu auto-check; nếu không đủ số thì không auto-check.
- Lỗi vẫn được lưu hidden dạng REJECTED_PLAN để phục vụ learning/debug, không xuất hiện trong history/stats/dashboard.

Cập nhật Sonnet Analyst Mode:
- AI bây giờ được quyền chọn NO_TRADE nếu chưa có setup rõ, thay vì bị ép phải ra LONG/SHORT.
- Bot gửi thêm market regime và raw candle context cho AI để đọc hành vi giá tốt hơn.
- Nếu AI chọn NO_TRADE, bot hiển thị phản hồi NO_TRADE của AI cho user; NO_TRADE không auto-check.
- NO_TRADE được lưu ẩn để học lại, không hiện trong /history, /stats, /dashboard và không auto-check.


Ghi chú bản completion guard:
- Bot không dùng Python risk/format guard để sửa hoặc chặn lệnh Claude.
- Nếu phản hồi Claude bị cụt rõ ràng, bot gọi Claude trả lại bản hoàn chỉnh; Python không tự viết hay sửa Entry/SL/TP.
- Có thể chỉnh CLAUDE_MAX_TOKENS trong Railway, mặc định 5000.


Ghi chú model AI:
- Admin có thể đổi provider trên Railway Variables.
- Claude mặc định: AI_PROVIDER=anthropic.
- Test GLM 5.2 rẻ hơn qua OpenRouter: AI_PROVIDER=openrouter, OPENROUTER_MODEL=z-ai/glm-5.2.
- Phân tích chính dùng OPENROUTER_REASONING_EFFORT=xhigh; nếu dùng GLM mà không khai báo biến này, code mặc định xhigh.
- Summary mặc định tắt reasoning: OPENROUTER_SUMMARY_REASONING_EFFORT=off, LLM_SUMMARY_MAX_OUTPUT_TOKENS=600.

Ghi chú OpenRouter/GLM:
- Nếu dùng GLM 5.2 qua OpenRouter, bot mặc định dùng reasoning xhigh cho phân tích chính nếu bạn không set OPENROUTER_REASONING_EFFORT.
- Railway log sẽ hiện call_type=main cho phân tích chính và call_type=summary cho tóm tắt learning để dễ đọc chi phí.
- Lịch sử học lại vẫn giữ 5 phân tích gần nhất của chính user cho cùng coin/mode.
